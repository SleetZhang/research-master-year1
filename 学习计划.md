## 一、调研综述：2024-2026 多模态后训练与 Agent 发展脉络

### 1.1 基础架构演进：从 BLIP 到 GPT-4o 时代

2023 年末至 2024 年，多模态大模型经历了从探索期到成熟期的快速跃迁。**LLaVA 原始论文**（NeurIPS 2023）开创了用纯文本 GPT-4 合成多模态指令数据的范式：将图像的 caption/边界框文字化后喂给 GPT-4，生成高质量对话数据，再用"CLIP 视觉编码器 + 线性投影 + Vicuna"的极简三件套完成端到端微调。这个框架的优雅之处在于：以极低成本证明了"多模态能力主要来自数据而非复杂架构"。

随后的 **LLaVA-1.5**（CVPR 2024）将线性投影升级为两层 MLP、视觉编码器换用 336px CLIP，并混入学术 VQA 数据，以不到 1% 的训练成本超越了当时所有开源模型。这确立了"MLP 投影 + 高分辨率 CLIP + 混合数据"的标准范式，被此后几乎所有工作沿用。

在架构演进上，**BLIP-2**（ICML 2023）引入了 Q-Former 两阶段预训练桥接冻结的视觉编码器和 LLM，以"冻结双端、只训中间桥梁"的方式实现参数高效的多模态对齐；**InstructBLIP** 在此基础上将 Q-Former 的提取过程条件化于指令，使视觉 token 提取随任务需求自适应调整。这条 Q-Former 路线与 LLaVA-MLP 路线并行发展，代表了连接器设计的两大思路。**MM1**（ICML 2024，Apple）则以少见的严格消融视角回答了"什么才是关键"：通过对 600M–30B 规模的 8+ 模型系统实验，MM1 发现连接器类型（MLP vs attention pooling vs conv等）对下游任务影响微乎其微，真正影响预训练质量的是视觉编码器选择和图文交错数据混合比例。这一结论与 Cambrian-1（"多编码器融合更好"）形成有趣的张力，需要结合各自的实验设定对比理解。

**架构多元化**在 2024 年形成清晰的技术谱系：
- **InternVL 系列**（CVPR 2024 Oral → InternVL 2.5）选择了"大视觉编码器（6B InternViT）+ 渐进式对齐"路线，将视觉编码器规模化到前所未有的程度，证明视觉侧扩展与语言侧扩展同样有效；
- **Qwen2-VL**（2024）引入 Naive Dynamic Resolution，视觉 token 数量随图像分辨率动态调整（而非 AnyRes 的固定切片），配合 M-RoPE 二维位置编码，大幅提升了文档、OCR 等精细视觉任务的性能；
- **Cambrian-1**（NeurIPS 2024 Oral）则以系统比较的视角发现：单一 CLIP 编码器是多模态理解的瓶颈，DINOv2 能补充 CLIP 缺失的结构信息，多编码器融合 + Spatial Vision Aggregator（SVA）是更优解；
- **Molmo**（NeurIPS 2024 Keynote）走了完全不同的路：Allen AI 用人工语音口述的方式收集高密度图像描述，证明在相同架构下，人工高质量数据完全可以超越 GPT-4V 合成数据。

### 1.2 多模态后训练：从 PPO 到 GRPO 的偏好对齐进化

**第一阶段（2023 末）：建立范式**

多模态 RLHF 的最早系统实践是 **LLaVA-RLHF**（ICLR 2024），它发现直接把文本 RLHF 搬到多模态会出现"奖励 Hacking"——奖励模型被语言流畅度欺骗，不奖励视觉忠实性。解决方案是"Factually Augmented 奖励"：在奖励模型输入中加入图像描述和 ground-truth，让奖励信号锚定事实。同年的 **RLHF-V**（CVPR 2024）则走了另一条路：收集段级别的人工纠错反馈（直接标注哪个句子是幻觉并修正），转化为 Dense DPO，仅 1.4K 样本减少幻觉 34.8%，超越 LLaVA-RLHF 的 10K 数据+PPO 效果。

**第二阶段（2024 上半年）：AI Feedback 规模化**

人工标注的瓶颈催生了 AI feedback 路线。**Silkie/VLFeedback**（2023 末）最先实践：从 12 个开源 LVLM 生成 82K 回答，用 GPT-4V 多维度打分，再 DPO 蒸馏。**mDPO**（EMNLP 2024）则深挖了一个关键问题：直接在多模态场景用标准 DPO 往往不稳定，原因是"无条件偏好问题"——DPO 优化的主要是文本语言偏好，模型可能学会忽略图像。mDPO 的解决方案是同时优化图像条件偏好 + 奖励锚定，成为此后多模态偏好对齐的重要参考。

**第三阶段（2024 末-2025）：自我对齐与 RL 推理增强**

**RLAIF-V**（CVPR 2025）的核心贡献是"分治法 AI 自我对齐"——将复杂的整体回答评估分解为简单的单声明验证，使开源模型也能可靠地充当 judge，最终让 12B 模型的可信度超越 GPT-4V，且无需任何人工标注或闭源 API。**Calibrated Self-Rewarding**（NeurIPS 2024）则在自我奖励中引入 CLIP 视觉校准，解决了纯文本自我奖励的视觉盲点问题。

2024 年末至 2025 年，**推理增强**成为多模态后训练最热的新方向，直接受 DeepSeek-R1 启发：
- **LLaVA-CoT**（ICCV 2025）首先将 o1 风格推理引入多模态，四阶段结构（摘要→描述→推理→结论）显式分离视觉感知和逻辑推理；
- **AtomThink** 将推理链分解为原子步骤并训练步骤级 PRM，在 MathVerse 上实现 120% 的相对提升；
- **Vision-R1**（2025）第一个系统性地将 GRPO + 可验证奖励应用于多模态，7B 模型在 MathVista 上与 OpenAI o1 持平；
- **Visual-RFT**（ICCV 2025）将 GRPO 扩展到视觉感知任务（检测、分类），用 100 个样本的 RL 微调超越 500 个样本的 SFT，证明 RL 的数据效率优势；
- **VisualPRM** 构建了 400K 的多模态过程监督数据集，PRM + BoN 的测试时计算使 MathVista 提升约 12 个百分点；
- **Mulberry**（NeurIPS 2025 Spotlight）用集体 MCTS（Collective Monte Carlo Tree Search）协同多模型搜索正确推理路径（成功率 80.2% vs 单模型 58.2%），构建出含丰富推理树的 Mulberry-260K 数据集，显示离线搜索+SFT 路线与在线 GRPO 路线各有优势。

在偏好对齐侧，**MM-RLHF**（2025）是目前最系统的多模态 RLHF 研究：120K 细粒度偏好数据 + 基于批判的奖励模型（Critique-based RM）+ 动态奖励缩放（Dynamic Reward Scaling），在 11 项基准上均优于 InstructBLIP 和 LLaVA-1.5。其核心洞察是"批判先于打分"——让奖励模型先生成详细批评再打分，显著提升了奖励质量。

### 1.3 多模态幻觉：从诊断到系统缓解

幻觉研究从"被动发现"走向"主动系统化"。评测侧，**POPE**（EMNLP 2023）用 Polling-based 二分类方式（问"图中有没有 X"）建立了对象幻觉评测的统一协议，后续几乎所有工作都报告 POPE 分数；**HallusionBench**（CVPR 2024）则进一步设计了视觉幻觉与语言幻觉交织的复杂诊断场景。**幻觉综述**（2024）从全景视角梳理了四类幻觉（对象/属性/关系/知识）和三类成因（语言先验偏差、视觉信息提取不足、训练分布偏差）。

缓解路线上，2024 年出现了两条清晰的技术路径：

**路径一：解码时干预（训练无关）**——通过修改生成时的 logit 分布来放大视觉信号、压制语言偏差。**VCD**（CVPR 2024）对比"有图"和"加噪图"的输出分布，将差值作为视觉对比信号；**OPERA**（CVPR 2024 Highlight）则发现"summary token 过度信任"是幻觉的根本注意力模式，通过在 beam search 中惩罚该模式实现无训练幻觉缓解。这类方法的优势是即插即用，劣势是推理开销增加（需额外前向传播）。

**路径二：训练时对齐**——通过数据和训练目标的修改从根本上消除幻觉倾向。DPO 系列（RLHF-V、mDPO）直接用偏好数据对齐；**HACL** 从对比学习视角调整视觉-文本表示空间的对齐质量；**HII-DPO**（2026）则专门针对"场景条件幻觉"——通过构建反事实图像（将目标对象替换为相似干扰物）创建幻觉-正确回答对，再做 DPO，精准消除"见过太多 XX 场景所以猜 XX"的模式。此外，推理增强（LLaVA-CoT、Vision-R1）通过强迫模型先"看清楚"（描述阶段）再推理，间接减少了推理幻觉。

### 1.4 多模态 Agent：工具调用与知识增强

2024-2025 年的多模态 Agent 研究逐渐从"通用能力"聚焦到三个具体机制：

**(1) 工具调用**：MLLM-Tool 建立了基础的多模态工具调用框架；Multi-modal Agent Tuning（MAT）自动生成工具调用数据做 SFT；**Visual-ARFT** 用 GRPO 训练 Agent 的工具调用序列，在多步任务上显示出 SFT 不具备的泛化优势。**AvaTaR**（NeurIPS 2024）则另辟蹊径：不用梯度更新，仅通过对比成功/失败轨迹生成工具描述优化建议，让 Agent 在 10-20 次迭代内自主改善工具调用策略，对没有训练数据的新工具尤其实用。

**(2) 记忆机制**：**HippoRAG**（NeurIPS 2024）从认知科学借鉴海马体的"开放书本"记忆机制，将知识图谱作为人工突触连接，用 Personalized PageRank 检索特定化事实，在多跳问答上比传统 RAG 提升显著；**MemVerse** 在此基础上扩展到多模态场景，构建双层层次化记忆（工作记忆+长期记忆），支持跨会话的多模态知识积累。

**(3) 视觉 RAG**：**VisRAG**（ICLR 2025）将 RAG 框架完全视觉化，文档页面以图像形式直接检索再由 VLM 生成，在复杂排版文档上比文字 RAG 提升 20-40%；**ColPali**（ICLR 2025）则从检索效率入手，将 ColBERT 的晚期交互（late interaction）范式扩展到多模态，用 patch 级别多向量表示文档页面，无需 OCR 即可高效检索。**MRAG-Bench**（ICLR 2025）揭示了视觉 RAG 的真正瓶颈：即使检索正确，VLM 也常常"检索对了但没用好"（检索召回率 vs 利用率的 gap），指出提升 VLM 视觉信息利用能力（后训练层面）是下一步关键。

### 1.5 为什么加入模块 0：RL 基础补漏

模块 2 的核心论文（LLaVA-RLHF、mDPO、RLAIF-V、Vision-R1、Visual-RFT 等）分属三条算法路线：PPO 路线、DPO 路线、GRPO 路线。这三条路线都以强化学习的基本概念为前置，强行跳过会反复卡在"为什么这么设计"上。

**演进关系**：PPO 是根 → DPO 绕开 PPO（用闭式解替代迭代优化）→ GRPO 简化 PPO（去掉 critic 网络，用 group-relative advantage 估计代替 GAE）。

**模块 0 内容**：PPO 教程（非论文，1 天）→ InstructGPT 精读（RLHF + PPO 完整落地，12h）→ DPO 精读（从 RLHF 目标到闭式 loss 的数学推导，12h）→ DeepSeek-R1 粗读（GRPO 原理，3h）。约 1 周（35h），安排在 Week 1，是模块 2 的必读前置。

**两个"都保留"决策的显式理由**

**VisRAG（M3-06）与 ColPali（M3-10）都保留**：两者代表视觉文档检索的两条完全不同的技术路线。VisRAG 是**端到端 VLM 路线**——文档页面以图像形式直接输入 VLM，检索与生成在同一模型内完成，无需 OCR 或文本抽取，优势是保留排版信息、处理复杂版面；ColPali 是 **ColBERT 风格的 patch 级多向量晚期交互路线**——用 VLM 为文档页面生成 patch 粒度的多向量表示，query 与每个 patch 向量分别计算相似度后聚合，优势是检索效率高（可预计算索引）且可解释。两者是**检索范式（多向量索引 vs 端到端生成）的互补对比**，不能互相替代。

**HippoRAG（M3-09）与 MemVerse（M3-05）都保留**：两者解决不同层次的记忆问题。HippoRAG 是**单模态（文本）长期记忆的架构基础**——借鉴海马体的开放书本联想机制，用知识图谱作为人工突触、Personalized PageRank 实现灵活检索，为多跳问答提供概念框架；MemVerse 是**多模态双层记忆框架**，在 HippoRAG 的记忆机制思想上扩展到视觉-语言场景，支持跨模态的工作记忆+长期记忆管理。**HippoRAG 是概念前置，提供记忆架构的认知基础，MemVerse 是其多模态延伸**，先读 HippoRAG 再读 MemVerse 有明显的理解梯度。
